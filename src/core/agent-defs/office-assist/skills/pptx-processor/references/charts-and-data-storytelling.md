# Charts and Data Storytelling

Start with the question the chart must answer. Use a table when exact lookup matters more than pattern recognition.

## Choose the form

- Use a line chart for change over continuous time.
- Use bars for comparison across categories; use horizontal bars for long labels.
- Use a scatter plot for relationships and outliers.
- Use a histogram or box plot for distributions.
- Use a waterfall for additive drivers when the arithmetic is important.
- Use maps only when geography is analytically relevant.
- Avoid pies with many categories and avoid all 3D charts.

## Focus attention

- Highlight the series, period, or category that supports the slide's main observation.
- Mute context without hiding it.
- Use direct labels when they remove legend lookup.
- Annotate meaningful inflections, gaps, events, and assumptions.
- Keep gridlines, ticks, borders, and decimals only when they help interpretation.

## Preserve analytical honesty

- Label units, period, source, filters, and whether values are actual, forecast, indexed, or estimated.
- Use comparable scales and disclose breaks or truncation.
- Avoid dual axes unless the relationship cannot be shown more clearly another way.
- Separate correlation from causation and fact from forecast.
- Show uncertainty, sample size, or missing data when material.

## Handle tables on slides

- Limit the table to rows and columns needed for the point.
- Align numbers by place value and use consistent precision.
- Highlight one reading path rather than coloring every cell.
- Move detailed source tables to an appendix or companion workbook.

## Build a data slide

1. Write the intended observation in plain language.
2. Select the smallest visual that demonstrates it.
3. Add context required to interpret it correctly.
4. Title the slide with the supported takeaway.
5. Verify chart ranges and displayed values against the source data.
