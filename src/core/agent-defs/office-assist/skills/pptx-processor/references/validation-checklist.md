# Presentation Validation Checklist

Validate both the file structure and the communication result.

## Structural integrity

- Reopen the PPTX without a repair warning.
- Confirm slide count, order, dimensions, masters/layouts, theme, notes, hyperlinks, and embedded media as applicable.
- Confirm no source slides or relationships disappeared during a targeted edit.
- Test video, audio, animation, fonts, and external links in PowerPoint when the task depends on them.

## Visual inspection

- Convert to PDF and use `to-images` for every slide, not only the title slide.
- Check overflow, overlap, clipping, off-slide objects, missing glyphs, image distortion, and font substitution.
- Check alignment, equal spacing, margins, contrast, and source-note legibility.
- Check charts for correct ranges, units, axes, labels, and emphasis.
- Check projection readability separately from close-up document readability.

## Sequence review

- Read only the slide titles and confirm they form a coherent outline.
- Confirm each slide advances the intended outcome and has one primary job.
- Check transitions, duplicated points, unsupported claims, unexplained acronyms, and missing decisions/actions.
- Check that detail is placed in the body, appendix, notes, or companion document appropriately.
- Check that composition varies enough to maintain attention without breaking the design system.

## Content and accessibility

- Verify names, dates, numbers, sources, units, and terminology.
- Ensure color is not the only carrier of meaning.
- Add descriptive labels or alternative text where the format and tooling support it.
- Confirm reading order and grouping are understandable for key diagrams.

## Delivery

- Preserve the original unless overwrite was requested.
- Remove unpacked folders, downloaded drafts, and temporary renders from the delivery set.
- Use clear filenames and include PDF or handout variants only when useful.
- State any element not verified in native PowerPoint.
